要在本地运行 QuickGraph，请按照以下步骤进行操作：

在 /server/.env 文件中的 TOKEN_SECRET 字段中添加一个安全令牌，用于用户密码的哈希和加盐处理。

在存储库的根目录中，打开终端并执行以下命令，使用 Docker 构建和运行 QuickGraph：

arduino
Copy code
$ make run
或者，您可以使用以下命令：

ruby
Copy code
$ docker-compose -f docker-compose.yml up

这将启动前端客户端（运行在端口3020）、后端服务器（运行在端口
3010）、MongoDB 数据库（运行在端口27018）和服务器集群（运行在端口8000）。验证所有
服务已正确启动后，您可以在浏览器中访问localhost:3020 开始使用QuickGraph

在 QuickGraph 中，您首先需要从项目仪表盘或页面通过点击“Annotate”按钮来进入注释视图。一旦进入，您可以通过点击或拖动数据集中的单词来选择它们，这将形成一个选中的文本跨度，选中的文本会以背景色变化表示。接着，您可以通过点击侧边栏的实体层级或使用标签搜索模态框来给这些选中的文本分配实体标签。例如，您可以将“Character”实体类型应用到文档中的“Alice”上。为了提高注释效率，QuickGraph 允许您使用实体传播功能，即通过点击已应用到“Alice”上的实体并选择“Apply All”图标，可以将“Character”实体类型传播到语料库中所有“Alice”的实例。在关系注释方面，您需要切换到关系模式，选择一个源实体和目标实体来创建它们之间的关系，如“Knows”。完成这些关系的创建后，记得点击保存图标，将数据集项标记为完成。这个过程帮助您高效地管理和标注数据集，从而构建出结构化的知识图谱。
